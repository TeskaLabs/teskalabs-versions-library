# LogMan.io v25.47 Release Notes

## Breaking Changes

- In Archive Explorer screen, only streams for the selected tenant are displayed to all users (including superusers).
- ECS schema is updated. For more details, see the [LMIO Common Library](#lmio-common-library) changelog.
- `lmio-admin` and `lmio-plus-admin` roles are removed and unified to a single role `*/admin`.

## Upgrade Procedure v25.30 â†’ v25.47

### Before the upgrade

- Roles `*/lmio-admin` and `*/lmio-plus-admin` have been obsoleted. Users with these roles must be assigned role `*/admin` instead.

### Upgrade

1. Change the version of LogMan.io in your `/Site/model.yaml` file to `v25.47`:

    ```yaml
    applications:
      - name: "ASAB Maestro"
        version: v25.47 # ...replace with the highest available patch
      - name: "LogMan.io"
        version: v25.47 # ...replace with the highest available patch
    ```

    Apply the changes to each node in the cluster one by one, consecutively.

2. Wait until all services are pulled and started, then Apply changes for the next node.

3. After all nodes have pulled and started the services, Apply the changes again (again one by one). This ensures that the latest versions of libraries used by LogMan.io services are pulled.

4. Wait until all services are started again.

### After the upgrade

1. Add the **LMIO Chart** service definition to your `/Site/model.yaml` file:

    ```yaml
    services:
        lmio-chart:
          - node01
          - node02
          - node03
    ```

    Apply the changes to each node in the cluster one by one, consecutively.

2. Wait until services are pulled and started, then Apply changes for the next node.

3. After all nodes have pulled and started the services, Apply the changes again (again one by one). This ensures that the latest versions of libraries used by LogMan.io services are pulled.

4. Wait until all services are started again.

The Discover screen requires at least the `/DataSources/events.yaml` file to exist in the `Library` in order to load data.
